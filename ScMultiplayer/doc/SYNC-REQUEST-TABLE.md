# ScMultiplayer 同步请求表

> 目标：所有会改变角色、实体或世界状态的交互均由主机校验和执行，客户端只负责发送请求与进行可回滚预测。
>
> 状态：`已覆盖` 表示已有完整请求或输入、主机执行、权威回传闭环；`部分覆盖` 表示只有输入或快照；`待实现` 表示尚无有效网络闭环。

## 一、统一协议规则

| 项目 | 规则 |
|---|---|
| 身份 | 使用传输层 `ClientID` 识别发送者，消息内的玩家编号不能覆盖传输层身份 |
| 顺序 | 操作请求携带 `Sequence` 和 `ClientTick`；主机按每个客户端去重，拒绝旧序列 |
| 权威 | 客户端不得直接决定伤害、掉落、实体生成、容器内容、投射物命中或爆炸结果 |
| 预测 | 移动、挖掘动画、手持切换可本地预测；主机拒绝时必须回传修正快照 |
| 校验 | 主机统一校验玩家存活、距离、视线、冷却、游戏模式、手持物、目标存在性和权限 |
| 目标 | 实体目标使用会话内稳定 `NetworkEntityId`，不能依赖本地 `Entity` 引用或列表下标 |
| 幂等 | 一次性操作使用 `RequestId`；相同请求重复到达时只执行一次，并重发原确认结果 |
| 返回 | 请求结果统一为 `Accepted`、`Rejected` 或 `Corrected`，拒绝时附带权威状态 |
| 频率 | 连续输入 100 Hz；角色状态按变化立即发送并以 4 Hz 保底；低优先级资料 1 Hz |
| 断线 | 主机清理运行时角色但保存角色记录；客户端清理下载世界、远程实体和待确认请求 |

## 八、实现顺序

| 阶段 | 内容 | 完成标准 |
|---|---|---|
| P0-1 | 投射物、命中、爆炸闭环 | 弓箭、投掷物和炸药只由主机生成结果，客户端无重复实体或伤害 |
| P0-2 | 箱子、熔炉、工作台容器事务 | 两端同时操作不会复制或丢失物品，拒绝旧版本事务并返回修正快照 |
| P0-3 | 动物专用攻击请求接线 | 客户端射线与主机动物位置有小偏差时仍可由主机合法命中，且一次请求只伤害一次 |
| P1-1 | 告示牌、家具、手持物编辑结果 | 对话框提交后由主机验证并广播最终内容 |
| P1-2 | 睡眠、着火、流感、生病 | 状态开始、持续、结束和视觉效果均以主机为准 |
| P1-3 | 精确坐骑 ID | 多只坐骑相邻时两端始终绑定同一实体 |

## 九、验收用例

| 用例 | 期望结果 |
|---|---|
| 客户端连续攻击玩家和动物 | 每次合法攻击只结算一次，血量、击退、硬直和仇恨一致 |
| 100–300 ms 延迟下移动、跳跃、蹲边 | 客户端无频繁回退，主机不允许超过物理上限，蹲下不会滑落 |
| 两端同时挖掘同一方块 | 方块只销毁一次，掉落物只创建一次，工具耐久一致 |
| 客户端从背包或箱子移动物品 | 主机确认前不被旧快照覆盖，确认后两端槽位和数量完全一致 |
| 两端同时打开同一箱子 | 旧版本事务被拒绝，不复制物品，并返回最新容器快照 |
| 客户端发射弓箭后退出重进 | 主机仍维护投射物；客户端重进只创建一个权威投射物 |
| 主机触发爆炸 | 两端伤害、冲量、火焰、烟雾和地形变化一致 |
| 客户端编辑告示牌或家具 | 主机保存最终结果，其他客户端立即看到，重进后仍存在 |
| 客户端睡觉或感染状态效果 | 主机角色状态与客户端界面一致，离开重进后按主机记录恢复 |
| 主机异常终止 | 客户端在心跳超时内退出世界并清理下载地图及远程实体 |
