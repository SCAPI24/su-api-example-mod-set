# Task: ScMultiplayer 联机代码完善 v1.0.2

**日期**: 2026-05-17 22:55
**目标**: 基于前次分析结果，完善局域网自动发现、世界存档下载、套用 StateMachine 优化网络流程，消息装箱拆箱规范化，产出可运行 .scmod

## 关键技术决策

### 1. 状态机集成
- 复用 Survivalcraft 自带的 `Game.StateMachine` (非新增)
- 两层状态机: NetworkConnectionStateMachine (连接生命周期) + WorldDownloadStateMachine (世界下载)
- 连接流: Disconnected → Discovering → WaitingForWorld → WorldDownloading → Playing

### 2. 消息系统完善
- 统一使用 `Message.WriteWithSender(msg, client.Address)` 替代 `Message.Write(msg, client.Address)` — 确保 IPEndPoint 序列化一致
- 新增 2 个 Message 类型: GamePlayerHealthMessage, GameKickPlayerMessage
- 反射注册基于字母序 ID 分配, 新类自动接入

### 3. 生命值同步
- ComponentHealth.Health 是 `{ get; private set; }` 只读属性
- 同步方案: 用 `health.Heal(diff)` / `health.Injure(-diff)` 替代直接设值
- 30fps 周期检查变化后才发送

### 4. 编译问题解决
- `ReadOnlyList<T>` 是 struct (非 class), `?.` 生成 `Nullable<ReadOnlyList<T>>`, 不支持 LINQ 扩展
  - 修复: 先存 Subsystem 变量判空, 再取 `.ComponentPlayers`
- `Server.Stop()` 不存在 → `Server.Dispose()`
- `ComponentHealth.HealthBarMax` 不存在 → 硬编码 1f (游戏默认最大值)
