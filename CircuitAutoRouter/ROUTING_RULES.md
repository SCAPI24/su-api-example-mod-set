# 电路自动布线规则 (Circuit Auto-Routing Rules)

> 基于 CircuitAutoRouter Mod 项目与用户的多次交互讨论确认的布线规则。
> 错误分数记录：6分

---

## 一、术语表

### 1.1 空间

**方格(Cell)**：游戏世界的1×1×1立方体，由3D整数坐标(x, y, z)定位。
俯视图中：col=X轴，row=Z轴，所有方块在同一Y层。
编号规则：从左到右、从上到下，左上角为1，右下角为16。

**面(Face)**：每个方格6个面，编号与方向：

| Face | 用户坐标 | 俯视图 | 3D向量(代码) | 对面 |
|------|---------|--------|-------------|------|
| 0 | -Z | 下(行号+1) | (0,0,1) | Face 2 |
| 1 | +X | 右(列号+1) | (1,0,0) | Face 3 |
| 2 | +Z | 上(行号-1) | (0,0,-1) | Face 0 |
| 3 | -X | 左(列号-1) | (-1,0,0) | Face 1 |
| 4 | +Y | 上方(Y+) | (0,1,0) | Face 5 |
| 5 | -Y | 下方(Y-) | (0,-1,0) | Face 4 |

对面映射：0↔2, 1↔3, 4↔5（代码：`OppositeFace = {2, 3, 0, 1, 5, 4}`）

**注意**：用户坐标系中Z轴正方向与代码向量相反。俯视图中Face0=下、Face2=上是确定的。

**CellFace**：`(x, y, z, face)` — 精确定位方格的某一个面。

**方格面坐标（关键）**：方格(x)的范围是[x, x+1]，面的平面位置：
- Face1(+X/右)在x+1处，Face3(-X/左)在x处
- Face0(+Z/下)在z+1处，Face2(-Z/上)在z处
- 同col方格的Face1/Face3在同一平面；同row方格的Face0/Face2在同一平面

### 1.2 方块类型

**导线块(WireBlock, Index=133)**：
- ≠实体方块，贴附在方格面上的薄层
- data & 0x3F = 6-bit位掩码(bit i=1 → Face i有导线点)
- 导线点位置：贴在方格面表面上
- 支撑要求：导线点邻面必须是实体方块

**穿线块(WireThroughBlock)**：
- =实体方块 ✓，占据整个1×1×1方格空间
- data & 3决定轴向：0→Z轴(F0↔F2), 1→X轴(F1↔F3), 2→Y轴(F4↔F5)
- 连接点位置：在面中心
- 只连通一组对面，其他面不导电

**实体性质对照**：

| 类型 | 实体 | 支撑导线 |
|------|------|---------|
| 穿线块 | ✓ | ✓ |
| 普通方块 | ✓ | ✓ |
| 导线块 | ✗ | ✗ |
| 空气 | ✗ | ✗ |

### 1.3 电路概念

**导线点**：导线块某面上的点，贴在面表面。
**穿线连接点**：穿线块面的中心位置。
**导线域(WireDomain)**：连通导线共享电压(OR逻辑)。
**查找表(ElectricConnectionPath)**：120项预计算，含正面和对角路径，连接检查双方GetConnectorType≠null即可。

### 1.4 布线概念

**基本块**：寻路算法确定路径后，路径上每个节点对应一个基本块。基本块位置固定，布线规则在此基础上决定每个基本块的方块类型和补导线点。

**补导线点**：Z干涉处理后，在穿线块导通反向端旁空位补的导线点，仅1个面，贴穿线块面。

**Z干涉**：两个紧贴方格都做导线块，导线点面对面缺实体支撑，连接错误。

**点变线**：同平面导线点自动延伸连通。仅导线点和导线点之间才触发，穿线连接点不参与点变线。

**田字**：四个方格构成2×2区域(A|B/C|D)，十字交叉处面也是相邻面。

## 二、布线规则

### 规则1：铁律 — 导线点实体支撑

**导线点所附加的面，其相邻方格必须是实体方块。**
违反 → 导线点被系统自动移除 → 电路连接错误。

### 规则2：方块类型选择

| 进出面关系 | 选择 |
|-----------|------|
| 对面(如Face1↔Face3) | **穿线块**(选择对应轴向) |
| 非对面(转弯) | **导线块**(在两个面放导线点) |

### 规则3：Z干涉判定

两个紧贴方格都做导线块 → 导线点面对面缺实体支撑 → Z干涉。
适用于所有轴向（上下、左右、前后）。

### 规则4：Z干涉处理（按路径顺序，先处理再考虑连接）

1. 按路径顺序逐个创建方块类型
2. 遇到两个导线块紧贴 → Z干涉
3. 将**后一格**改为穿线块（**导通方向朝下一个基本块**）
4. 在穿线块**导通反向端**旁空位补**1个导线点**，贴穿线块面
5. **补导线点只需1个面！不创建额外面来桥接！**

**关键**：先做处理，再考虑连接！穿线块导通方向由下一个基本块位置决定，不由前一个基本块决定。

### 规则5：点变线 — 导线仅和导线才让点变线

**穿线块的连接点（面中心）不会被导线点自动延伸连通。**

- 同平面导线点自动延伸连通（"点变线"）— 只连导线点，不连穿线连接点
- 点变线距离 = 相邻面，如果下一格也需要点变线，需要一个新的点
- **但查找表连接不受此规则限制**：正面相邻方格的面通过查找表连通，不管导线vs穿线

### 规则6：田字连通 — 十字处面也是相邻面

田字(如A|B/C|D)的十字交叉线处，所有面在同一平面：

```
 A | B       十字在z=A.z+1和x=A.x+1处
 -----       z=A.z+1: A Face0, B Face0, C Face2, D Face2
 C | D       x=A.x+1: A Face1, C Face1, B Face3, D Face3
```

十字处面连通：
- A Face0↔C Face2 [正面相邻] ✓
- B Face0↔D Face2 [正面相邻] ✓
- A Face1↔B Face3 [正面相邻] ✓
- C Face1↔D Face3 [正面相邻] ✓
- B Face0↔C Face2 [同z平面, 十字相邻, 点变线] ✓
- A Face1↔C Face1 [同x平面, 十字相邻, 点变线] ✓

**补导线点只需1个面**，通过十字相邻点变线同时接收和传出信号：
- 接收信号：十字相邻点变线
- 传出信号：查找表正面路径

**不要为补导线点创建额外的面**，那是废弃线路。严格按照规则4：只需1个面。

### 规则7：死端导线点

创建过程中产生的多余导线点不影响电路运行，不需要额外删除。

### 规则8：导线内部连通

- 本面有导线→标记自身
- 非对面也有导线→flag=true→标记那些面
- flag=true且对面也有导线→标记对面
- **仅两面(对面)有导线不直接连通**，需要中间面桥接

**实际用途**：匹配后用穿线块替代，在布线算法中不主动利用此机制。

## 三、错误记录与教训

| # | 错误 | 教训 |
|---|------|------|
| 1 | 忽略"导线不连穿线"规则 | 点变线只连导线点，不连穿线连接点；但查找表连接不受此限制 |
| 2 | 坐标计算错误(3的Face3在x=2而非x=1) | 方格(x)范围是[x,x+1]，同col方格同方向面在同一平面 |
| 3 | 忽略田字十字相邻面规则 | 田字十字处的面也是相邻面，点变线可互相连通 |
| 4 | 为补导线点创建额外的面 | 补导线点只需1个面，不创建额外面来桥接（否则是废弃线路） |
| 5 | 修改基本块位置 | 基本块由寻路算法确定，不可修改；只能在基本块基础上决定类型和补点 |
| 6 | 忘记规则4"导通方向朝下一个基本块" | 穿线块轴向由下一个基本块位置决定，不由转弯方向决定 |
| 7 | 混淆内部连通和创建的导线面 | 导线块只有显式创建的面有导线点，内部连通是标记机制不是创建新面 |
| 8 | 不参考规则就执行 | 每次执行前必须回看规则，规则是执行依据 |

## 四、完整布线示例

### 4.1 示例1：简单Z转弯

路径 1(Face3进) → 8(Face1出)：
```
1  2  .  .
.  6  7  8
```

| 方块 | 类型 | 说明 |
|------|------|------|
| 1 | 穿线X | 左进右出，对面 |
| 2 | 导线(上,右) | 转弯 |
| 5 | 导线(右) | 补点贴6左面 |
| 6 | 穿线X | 左进右出 |
| 7 | 穿线X | 左进右出 |
| 8 | 穿线X | 左进右出 |

### 4.2 示例2：田字Z干涉

路径 2(Face2进) → 16(Face1出)，基本块：2,6,7,11,15,16

按路径顺序逐个创建：
1. 2：上进下出→对面→穿线Z
2. 6：上进右出→转弯→导线(上,右)
3. 7：左进下出→转弯→导线(左,下) → 与6导线紧贴→Z干涉！

Z干涉处理（规则4）：
- 7改为穿线Z（导通方向朝下一个基本块11，即下方）
- 导通反向端=F2(上)，旁空位=3号位
- 3补导线F0(下)贴7穿线F2(上)，仅1个面

| 方块 | 类型 | 说明 |
|------|------|------|
| 2 | 穿线Z | 上进下出 |
| 6 | 导线(上,右) | 转弯 |
| 3 | 导线(下) | 补点贴7穿线上面，仅1个面 |
| 7 | 穿线Z | Z干涉改，导通朝11(下方) |
| 11 | 穿线Z | 上进下出 |
| 15 | 导线(上,右) | 转弯 |
| 16 | 穿线X | 左进右出 |

连通验证：
- 2穿线F0→6导线F2[正面相邻]✓
- 6导线F2(上,z=1)和3导线F0(下,z=1)同z平面→点变线✓
- 3导线F0→7穿线F2[正面相邻,查找表]✓
- 7穿线内部F2↔F0→7穿线F0→11穿线F2[正面相邻]✓

6导线F1(右)贴7穿线F3(左)是死端(7穿线Z的F3不导电)，不影响。

### 4.3 示例3：3处Z干涉

路径 1(Face3进) → 9(Face3出)，基本块：1,2,3,7,8,9,12,13,14,15,16

按路径顺序逐个创建：
1. 1：左进右出→对面→穿线X
2. 2：左进下出→转弯→导线(左,下)
3. 3：上进右出→转弯→导线(上,右)→与2导线不紧贴(row0 vs row0,col2 vs col1)

Z干涉处理：
1. 3和7 → 3导线F0(下)和7导线F2(上)紧贴→7改穿线X(导通朝8=右方)，6补导线F1(右)
2. 7和8 → 7已是穿线X=实体，8无干涉✓
3. 13和9 → 13导线F1(右)和9导线F3(左)紧贴→9改穿线X(导通朝12=右方)，10补导线F3(左)

### 4.4 示例4：田字十字连通

路径 3(Face2进) → 5(Face3出)，基本块：3,7,8,12,5

7和8紧贴→Z干涉→8改穿线Z(导通朝12=下方)，4补导线F0(仅1个面)

3|4/7|8田字十字连通：
- 3穿线F0(下)→7导线F2(上)[正面相邻]✓
- 7导线F2(z=1)和4导线F0(z=1)[十字相邻,点变线]✓
- 4导线F0→8穿线F2[查找表正面路径]✓

7导线F1(右)是转弯时创建的死端，不影响。
不需要4的F3（那是废弃线路）。

## 五、已确认问题

1. **点变线距离**：=相邻面。下一格也需要点变线→需要新点。
2. **导线内部连通**：实际用途是匹配后用穿线块替代，算法中不主动利用。
3. **查找表对角路径**：待确认对角路径是否也能导线连穿线。
